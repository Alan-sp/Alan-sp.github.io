4.CF666C Codeword

shiroi\color{black}\text{s}\color{red}\text{hiroi}shiroi Orz。

显然，答案只与 ∣S∣|S|∣S∣ 有关，具有普适性。因此我们有一个 naivenaivenaive 的预处理dp做法。

设 f[i][j]f[i][j]f[i][j] 表示现在的串的前 jjj 位包含了 SSS 的前 iii 位的方案数。

那么转移很显然，只要看当前位包不包含 SSS 的下一位即可。

f[i][j]=25f[i][j−1]+f[i−1][j−1]f[i][j]=25f[i][j-1]+f[i-1][j-1] f[i][j]=25f[i][j−1]+f[i−1][j−1]

这样做是 O(n2)O(n^2)O(n2) 的。仔细观察方程，发现它非常类似组合数的递推，或许我们可以想到更好的转移方式。

我们再考虑一下 f[i−1][j−1]f[i-1][j-1]f[i−1][j−1] 的实质：从前 j−1j-1j−1 位中选出 i−1i-1i−1 位填 SSS 中的字符，剩下的 “随便填”。

如果真的随便填的话，显而易见是会算重的，那么可以通过加一些约束来得到正确答案。

考虑重复的情况：当剩下的位“随便”填了 ∣S∣|S|∣S∣ 中的字符时，它在另一种方案中被选择。

思考匹配两个串的过程：对于两个相邻的 “匹配点”，我们钦定它们不能填后面那个 “匹配点” 的字符。

我们给这 (j−1i−1)\binom{j-1}{i-1}(i−1j−1​) 种方案规定顺序：按匹配点从右向左排序。

这样，在匹配点左移的过程中，他所到的位置在之前的方案中并没有填它的字符，也就不会算重了。

所以 f[i−1][j−1]=(j−1i−1)25j−if[i-1][j-1]=\binom{j-1}{i-1}25^{j-i}f[i−1][j−1]=(i−1j−1​)25j−i，我们找到了一种不依赖之前 iii 来转移的方式。

这样下来，每个 iii 之间是独立的，单次更新一个 iii 是 O(n)O(n)O(n) 的。

题目保证 ∑∣S∣⩽105\sum|S|\leqslant 10^5∑∣S∣⩽105，那么不同的 ∣S∣|S|∣S∣ 只会有 105\sqrt10^51​05 种。

直接暴力做就好，复杂度 O(nn)O(n\sqrt n)O(nn​)。

6.CF724F Uniformly Branched Trees

非常 有意思 的计数。

从重心考虑，钦定一棵树的重心为根，进行计数 dpdpdp。

这样子每棵子树的大小不超过 n2\frac n 22n​，并且方便去重。

由于要考虑不同构的情况，所以我们按子树的大小从小向大计算。

我们设 f[i][j][k]f[i][j][k]f[i][j][k] 表示当前有 iii 个节点的树，根节点的度为 jjj，子树大小不超过 kkk 的方案数。

那么显然有转移：f[i][j][k]=f[i][j][k−1]f[i][j][k]=f[i][j][k-1]f[i][j][k]=f[i][j][k−1]，不超过 k−1k-1k−1 的一定不超过 kkk 。

接下来我们就要考虑大小恰好为 kkk 的子树了。

枚举我们选了 ttt 棵大小为 kkk 的子树，那么剩下的方案数就是 f[i−tk][j−t][k−1]f[i-tk][j-t][k-1]f[i−tk][j−t][k−1]，注意剩下的子树大小不能超过 k−1k-1k−1。

而这 ttt 棵树有不同的形态，我们知道大小为 kkk，且合法的树的形态是 f[k][d−1][k−1]f[k][d-1][k-1]f[k][d−1][k−1]。

除去根节点，子树总大小为 k−1k-1k−1，并且因为这棵树要作为子树，根节点的度有一条是连向 fatherfatherfather 的。所以有 d−1d-1d−1 棵子树。

那么我们要从这么多种形态的树种选择 ttt 棵，同时这些树的形态可以重复，这就是一个可重组合，方案数为

(f[k][d−1][k−1]+t−1t)\binom{f[k][d-1][k-1]+t-1}{t} (tf[k][d−1][k−1]+t−1​)

在转移时还要乘上上面的组合数。

这样我们就处理完一个重心的问题了，但是还没有结束。因为当 nnn 为偶数时，可能这棵树有两个重心。

其他题解里对为什么会算重讲得不是很详细，这里我细说一下。

我们在考虑 树的同构 的时候，将子树的大小从小到大考虑，但这样真的考虑了所有情况吗？我们来看下面的图：

如上图，它们子树的大小的确是递增的，但是这两棵树却是同构的。

这是因为在两个重心分别当作根时，它与那些大小小于 n2\frac n 22n​ 的子树共同成的树形态是不同的，即去掉重心之间连边形成的两棵树形态不同。这样就会被重复计数。

统计这种情况，可以将两个重心的连边看成点，它连接着两棵大小为 n2\frac n 22n​ 的子树。

同上，大小为 n2\frac n 22n​ 的树形态有 f[n2][d−1][n2−1]f[\frac n 2][d-1][\frac n2 -1]f[2n​][d−1][2n​−1] 种，我们要从这些形态里选出两种不重复的去重。

这样的方案数是：

(f[n2][d−1][n2−1]2)\binom{f[\frac n 2][d-1][\frac n2 -1]}{2} (2f[2n​][d−1][2n​−1]​)

统计答案时减掉就好了。

注意特判 n⩽2n\leqslant2n⩽2 的情况。
